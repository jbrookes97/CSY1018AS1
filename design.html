Design and Good Practice There are many considerations when developing a website. Some of these may be ethical, some of them are regarding good practice. This page endeavours to provide a basic overview of the considerations that one may need to take whilst developing a website. There are some key things to think about whilst coding for a website, one that is often missed out is the thought “what if someone else needs to read my code”. If this was to be thought about, it may actually mean that some coding conventions actually become second nature to use throughout development. This would not only enable someone else to read our code, but also make it easier for us in the long run if for some reason we need to revisit the code. Things such as using meaningful class names and ids for the CSS can be very important. Adding comments throughout the CSS file can mean that when reading later it becomes far easier to understand what and why you may have designed it in this way. Furthermore, ensuring that all opening tags have a paired closing tag, or that they are initially self-closing (if applicable) ensures that all browsers will be able to interpret the code correctly. This section shall cover accessibility. The principle of accessibility with regards to software can be defined as providing access to all regardless of any situational or environmental circumstances. This roughly translates to doing all that you can to allow those less fortunate than ourselves all possible opportunities to access the sites. One of the first considerations from a design aspect with regards to web development will be the colour themes. One must ensure that the contrast is good so that the text is readable. A further expansion on this could be the use of themes to suit the needs of the three main kinds of colour blindness that visitors may be affected by. Another large consideration is the use of “ALT” text for images, if a developer consistently and correctly uses “ALT” tags throughout the development of websites it enables those without site to use additional software such as screen readers to access the content on the site. Adding “ALT” text to all media content enables the device to help to describe and therefore ensure they do not miss out from content such as images.
